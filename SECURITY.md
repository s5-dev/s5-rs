# Security Policy

## Supported Versions

| Version       | Supported          |
| ------------- | ------------------ |
| 1.0.x (beta+) | :white_check_mark: |
| < 1.0         | :x:                |

## Reporting a Vulnerability

We take the security of S5 seriously. If you believe you have found a security vulnerability in any S5 crate, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

### Private Reporting

You can report vulnerabilities via:

1.  **GitHub Security Advisories**: If enabled on the repository, please use the "Report a vulnerability" button in the Security tab.
2.  **Email**: Send details to `security@s5.pro`.

### What to Include

*   A description of the vulnerability.
*   Steps to reproduce the issue.
*   Potential impact.
*   Any relevant code snippets or proof-of-concept code.

### Response Timeline

We are committed to addressing security issues promptly. You can expect:

*   An acknowledgement of your report within 48 hours.
*   Regular updates on our progress.
*   A coordinated disclosure date.

Thank you for helping keep S5 safe!
